## CRITICAL Open Issues

### 1. Docker Agents Can't Reach bcd

**Problem:** `WriteWorkspaceHookSettings()` in `hooks.go:191` hardcodes:
```go
bcdAddr := "http://127.0.0.1:9374"
```

Docker agents POST hooks to `127.0.0.1:9374` which is localhost *inside the container*. Hooks fail silently (`|| true`).

**Impact:** Docker agents run but state never updates in bcd or web UI.

**Fix:** Add runtime parameter to `WriteWorkspaceHookSettings()`:
- tmux: `http://127.0.0.1:9374`
- Docker: `http://host.docker.internal:9374`

The Python script (`scripts/configure-hooks.py`) already handles this. The Go function needs the same.

### 2. `StopFailure` Still Generated by Go Code

`WriteWorkspaceHookSettings()` at `hooks.go` still generates a `StopFailure` hook entry. Claude Code rejects this key. It was manually removed from `.claude/settings.json` but will be re-added next time the function runs.

**Fix:** Remove the `"StopFailure"` line from the hooks map in `WriteWorkspaceHookSettings()`. Keep `HookStopFailure` in `hookEventStateMap` for the HTTP endpoint.

### 3. Root Agent Stuck After Startup

Likely causes:
1. If Docker: hooks can't reach bcd (issue #1)
2. If tmux: check `BC_AGENT_ID` env var is set, check `.claude/settings.json` exists in worktree
3. Hook scripts may not be configured in the agent's worktree

### 4. sharp-wolf Working But No Frontend Updates

Same root cause as #1 — Docker agent hooks can't POST to bcd.
