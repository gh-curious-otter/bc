How it works

1

Initialise a workspace

Run bc init in your project directory. bc creates a .bc/ directory with config, role definitions, and a SQLite database.

2

Start your root agent

Run bc up to start the root agent in a tmux session. The root agent has full workspace capabilities and can spawn a team.

3

Create specialised agents

Spawn engineers, managers, and QA agents with bc agent create. Each gets its own git worktree and role-based capabilities.

4

Coordinate through channels

Agents communicate via bc channel send. Use #eng, #pr, and #standup channels to keep work organised.

5

Monitor with bc home

Open the TUI dashboard with bc home to watch agents work in real time, review costs, tail logs, and manage the team.

Works with any AI agent

bc is agent agnostic. Use --tool to pick your preferred AI.

Quick start

Build from source

git clone https://github.com/rpuneet/bc
cd bc
make build
make install

Prerequisites

  • Go 1.22+
  • tmux
  • A configured AI agent tool (Claude Code, Cursor, etc.)

First workspace

# In your project directory
bc init
bc up
bc home
