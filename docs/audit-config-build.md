# Config and Build System Audit

## Overview

Audit of config/, scripts/, Makefile, and CI/CD for bead bc-34b.9.

## config/ Package

### Structure

| File | Purpose |
|------|---------|
| doc.go | Package doc with go:generate directive |
| config.go | Generated by cfgx from config.toml |

### Generated Config

The config is generated from `config.toml` using [cfgx](https://github.com/gomantics/cfgx):

```bash
go generate ./...
# or
make gen
```

**Config Sections:**
- `Agent` - Default agent command, coordinator name, worker prefix
- `Agents` - List of supported agents (claude, cursor, codex)
- `Costs` - Cost tracking settings (enabled, limits, thresholds)
- `Roles` - Role definitions with prompts and permissions
- `Tmux` - Session prefix settings
- `Tui` - Refresh interval and theme
- `Workspace` - State directory and max workers

### Issues

1. **No config validation** - Generated config doesn't validate values
2. **No runtime config override** - Can't override via environment variables or flags
3. **Hardcoded prompt paths** - Prompt files reference `prompts/*.md` but directory doesn't exist

## Recommendations

### High Priority (CI is broken)

1. **Add missing Makefile targets**: `coverage`, `build-release`, `build-all`
2. **Fix Go version**: Align go.mod and CI (use go 1.23 or update both)
3. **Run go mod tidy**: Fix the x/term dependency warning

### Medium Priority

1. **Add ldflags to build target**: Include version info in binary
2. **Create prompts/ directory**: Or remove prompt_file references from config
3. **Add config validation**: Validate generated config values

### Low Priority

1. **Add environment variable overrides**: Allow runtime config changes
2. **Add goreleaser**: For more robust release builds
3. **Add pre-commit hooks**: Enforce fmt/vet before commits
