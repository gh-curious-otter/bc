# Issue #1691: Embed Agent tmux Sessions in TUI - Feasibility Analysis

**Investigator:** eng-04
**Date:** 2026-02-24
**Status:** Research Complete

## Executive Summary

Embedding tmux sessions directly within the Ink/React TUI is **technically challenging** and **not recommended** for the current architecture. Better alternatives exist.

## Current Implementation

The TUI already provides agent output viewing via:

1. **Output Tab** (`AgentDetailView.tsx:146-156`):
   - Uses `bc agent peek <name> --lines 50`
   - Polls every 2 seconds

2. **Live Tab** (`AgentDetailView.tsx:159-175`):
   - Uses `bc agent peek <name> --lines 200`
   - Polls every 500ms
   - Supports j/k scrolling, follow mode, g/G jump

The Go backend supports this via `tmux.Manager.Capture()` which runs:
```
tmux capture-pane -t <session> -p [-S -<lines>]
```

## Challenges with True tmux Embedding

### 1. Nested tmux Sessions
- If user runs TUI inside tmux, embedding another tmux session creates nesting issues
- tmux control mode (`tmux -C`) could help but adds complexity

### 2. Ink/React Rendering Model
- Ink renders React components to ANSI terminal output
- Cannot embed raw PTY streams within React component tree
- Would require breaking out of Ink's rendering model

### 3. PTY Forwarding
- True embedding requires PTY (pseudo-terminal) forwarding
- Node.js can spawn PTYs via `node-pty` but Ink doesn't support inline PTY views
- Would need custom terminal emulator component (xterm.js-like for terminal)

### 4. Input Handling
- Keyboard input would need routing between TUI and embedded session
- Modal switching between TUI mode and passthrough mode

## Recommended Alternatives

### Option A: Enhanced Live Mode (LOW EFFORT - RECOMMENDED)
Improve the existing Live tab:
- Increase polling frequency to 100-200ms when active
- Add ANSI color preservation (currently stripped)
- Add wider capture (full terminal width)
- Add search within output

**Effort:** 1-2 days
**Impact:** High - addresses 80% of the use case

### Option B: Popup Attach (MEDIUM EFFORT)
Add an "attach" action that suspends TUI and attaches to tmux session:
- Press 'a' to attach to selected agent's tmux session
- TUI suspends, user gets full tmux session
- Detach returns to TUI

**Implementation:**
```typescript
// In AgentDetailView
if (input === 'a') {
  process.stdout.write('\x1b[?1049l'); // Exit alternate screen
  const { spawnSync } = require('child_process');
  spawnSync('tmux', ['attach-session', '-t', agentSession], { stdio: 'inherit' });
  process.stdout.write('\x1b[?1049h'); // Re-enter alternate screen
}
```

**Effort:** 1 day
**Impact:** High - provides full tmux access

### Option C: Split Pane Mode (HIGH EFFORT)
Use tmux pane splitting to show agent session alongside TUI:
- TUI creates a tmux split showing agent output
- Requires TUI to detect if running in tmux
- Complex pane management

**Effort:** 3-5 days
**Impact:** Medium - only works when TUI is in tmux

### Option D: True Embedding with xterm.js (VERY HIGH EFFORT)
Build custom terminal emulator component:
- Port xterm.js concepts to Ink
- Full PTY support within TUI
- Essentially building a terminal within a terminal

**Effort:** 2-4 weeks
**Impact:** High but over-engineered

## Recommendation

**Implement Option A (Enhanced Live Mode) + Option B (Popup Attach)**

1. **Phase 1**: Enhance Live tab with ANSI colors and faster polling
2. **Phase 2**: Add 'a' key to suspend TUI and attach to tmux session

This provides:
- Fast, near-real-time output viewing (Live mode)
- Full tmux access when needed (attach mode)
- Minimal complexity
- Works regardless of whether TUI is in tmux
